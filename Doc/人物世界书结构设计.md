# 人物世界书结构设计

> 与 [角色蓝图.csv](角色蓝图.csv) 配套的文件分组与路由设计。**100 名女角色** → **44 个绿灯条目文件**。
> 文件命名 / 路径约定：`世界书/凡界/<region>/人物/人物-<分组名>.txt`，与现有人物目录沿用同一规范。

---

## 一、设计原则

1. **同宗→同文件**：宗门/组织内所有成员合在一个文件，文件内部用 EJS 按角色名/道号路由输出。
2. **王朝公主细分**：不同凡国（甚至不同城市）拆成独立文件，留扩充空间。
3. **散修异士细分**：每个城市/邪派/族裔单独成文，哪怕暂时只 1 人；将来加新 NPC 直接进同文件。
4. **绿灯触发原则**：文件 yaml 关键字 = **内部所有角色名 + 道号 + 宗门/族/城/标志地标**。任一命中即激活全文件，EJS 内部决定输出哪位角色的 detail。
5. **顺序约定**：人物绿灯条目 `顺序: 56`（低于地区总览的 57/58 一档），让总览先布局再叠人物。

## 三、EJS 内部路由模板

每个文件统一遵循的输出逻辑：

```ejs
<%_ { _%>
<%_
// 1) 读 chat 与 stat_data，拼成扫描文本
const 具体地点 = getMessageVar('stat_data.地点.具体地点') || '';
let scanText = 具体地点;
try { scanText += '\n' + (getChatMessages(10) || []).join('\n'); } catch(e) {}
const mentioned = (kws) => kws.some(kw => kw && scanText.includes(kw));

// 2) 角色字典：每位一份 brief（一句话名片）与 detail（完整人设）
const C = {
  '白羽弦': {
    kws: ['白羽弦'],              // 个人触发关键字（多含本名+道号）
    brief: `- 白羽弦(女·化神中期·聆风斋斋主)：清音律承嗣、慈悲护道`,
    detail: `### 白羽弦
- 道号/职位：聆风斋斋主、清音律承嗣
- 境界：化神中期
- 外观：御姐 / 墨发盘髻素玉簪、淡紫眸、肤白胜玉
- 性格：心怀慈悲、把抚琴当治病救人...
- 着装：月白广袖琴袍、白绫齐胸...
- 关系网：...`
  },
  '林幽笛': { kws: ['林幽笛'], brief: `...`, detail: `...` },
  '顾流商': { kws: ['顾流商'], brief: `...`, detail: `...` },
};

// 3) 路由策略
const hitChars = Object.entries(C).filter(([n, c]) => mentioned(c.kws));
const sectKws = ['聆风斋', '清音律', '回音谷'];  // 文件级（宗门）关键字
const sectHit = mentioned(sectKws);

let outputText = '';
if (hitChars.length > 0) {
  // 命中具体角色 → 输出 detail
  outputText = hitChars.map(([n, c]) => c.detail).join('\n\n');
  // 若同时命中宗门 → 附其他成员 brief 作环境
  if (sectHit) {
    const others = Object.entries(C).filter(([n]) => !hitChars.find(([h]) => h === n));
    if (others.length) {
      outputText += '\n\n[同宗其他人物]\n' + others.map(([n, c]) => c.brief).join('\n');
    }
  }
} else if (sectHit) {
  // 仅命中宗门 → 输出全部 brief（人物名册）
  outputText = '[聆风斋人物名册]\n' + Object.values(C).map(c => c.brief).join('\n');
}
_%>
<%- outputText %>
<%_ } _%>
```

**输出三档**：
- **角色命中**（chat 提到本名/道号）→ 该角色 detail
- **宗门命中**（chat 只提宗门/地标）→ 全部成员 brief
- **角色+宗门同时命中** → 角色 detail + 其他成员 brief

## 四、yaml 配置模板

```yaml
- 名称: 人物-北境-聆风斋
  启用: true
  激活策略:
    类型: 绿灯
    关键字:
      # 全员本名
      - 白羽弦
      - 林幽笛
      - 顾流商
      # 道号/称谓（若有）
      # （聆风斋暂无道号）
      # 宗门名 + 功法/标志地标
      - 聆风斋
      - 清音律
      - 回音谷
  插入位置:
    类型: 角色定义之前
    顺序: 56              # 比地区总览(57/58)低一档
  激活概率: 100
  特殊效果:
    黏性: 5               # 命中后保持 5 轮、避免对话中途丢失人设
  递归:
    不可被其他条目激活: false
    不可激活其他条目: true
  文件: 世界书\凡界\北境\人物\人物-北境-聆风斋
```

## 五、命名 / 路径规范

| 项 | 规则 |
|---|---|
| 文件名 | `人物-<region>-<分组名>.txt` |
| 物理路径 | `世界书/凡界/<region>/人物/人物-<region>-<分组名>.txt` |
| yaml 名称 | 与文件名同名（去 `.txt`）|
| 触发顺序 | `56`（地区总览 57/58、人物 56、其他 ≤55） |

**`<分组名>`** 优先级：
- 大宗门名 → `聆风斋` / `玉女宗` / `万妖盟-潜龙族`
- 凡国/王朝名 → `沧澜王朝` / `楚天王朝` / `乾元皇室`
- 官署名 → `北辰学宫` / `镇魔司`
- 单一城市/势力 → `烬阳城` / `临渊城` / `净世军`
- 异族/特殊 → `碧落海族` / `雾林木灵`

## 六、扩充指南（将来加新人）

1. **新角色属于已有分组** → 直接打开对应文件，往 `C` 字典里加一条 `{kws, brief, detail}`；同时在 yaml `关键字` 里加该角色本名/道号。
2. **新角色属于已有大区但无对应分组**（如新城市 NPC）→ 新建 `人物-<region>-<新分组>.txt`，仿模板写一份。
3. **跨region 角色**（如某人物在两个 region 都活跃）→ 放在主要驻地的文件里，次驻 region 的 yaml 关键字也可挂上她的本名。
4. **男性人物**：未来若要补男角色，同样按"宗门→分组→文件"逻辑，命名为 `人物-<region>-<分组>.txt`（不区分性别，同一宗门男女在同一文件里更方便路由）。

## 七、与 [角色蓝图.csv](角色蓝图.csv) 的映射

蓝图 csv 是**人设池**，本结构是**世界书载体**。映射关系：
- csv 一行 = 一位角色（提供 关键字 / 初遇地点 / 外观 / 女性魅力 / 外观特征 / 穿衣习惯 / 性格特征 共 9 列）
- 世界书文件 `C[名字].detail` = 把那一行展开成完整人设段落
- 世界书文件 `C[名字].brief` = 那一行的浓缩名片
- 世界书 yaml `关键字` = csv 「关键字」列直接复用 + 该文件统一附加的宗门/地标 kws

填充新文件时直接对照 csv 行复制粘贴，把 9 列字段渲染进 detail 模板即可。
