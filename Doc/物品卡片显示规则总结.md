# 正文美化 · 物品卡片（item-card）显示规则总结

> 来源：`tavern_helper_template-main/src/面板美化/正文美化.html`
> 渲染函数：`renderItemCard(card)`（约 4364 行）、图标表 `ITEM_TYPE_ICON`（4190）、五行色 `ITEM_EL_COLOR`（4211）、品质色带（992-999）。

## 一、核心结论：所有类型共用同一张卡

正文美化里 **没有** 为每个大类/子类型做不同的卡片布局——`功法 / 装备 / 物品 / 傀儡 / 灵兽` 全部走同一个 `.item-card` 模板。类型之间的差异只体现在：

1. **左侧圆形图标**（按 `类型` 取 FontAwesome 图标）
2. **左边框色带 + 品质 pill 配色**（按 `品质`）
3. **哪些字段被填充**（如傀儡/灵兽才有资源条与技能；秘籍才有完整度/阅读进度）

也就是说，视觉区分是「图标 + 配色 + 出现的字段」的组合，而非独立版式。这正是当前初始资材剧情卡"不够有辨识度"的根源。

## 二、卡片结构（自上而下）

```
.item-card.ic-q-bg-<品质>        ← 左边框按品质染色
 ├ .ic-head                      ← 头部：圆形图标 + 名称 + meta pills
 │   ├ .ic-icon > i.<类型图标>
 │   └ .ic-headinfo
 │       ├ .ic-name              ← Cinzel 衬线，主强调色
 │       └ .ic-meta              ← [品质品][类型][境界][五行圆点][×数量][完整度][进度]
 ├ .ic-stats                     ← 数值 chip（攻击力/防御力/命中/…，按名着色）
 ├ .ic-resources                 ← 仅傀儡/灵兽：气血/灵气 进度条 + 遁速
 ├ .ic-tagrow                    ← 描述性标签（虚线 pill）
 ├ .ic-effects                   ← 效果块：左竖线，逐行「名: 描述」
 ├ .ic-skills                    ← 仅傀儡/灵兽：技能列表
 ├ .ic-desc                      ← 斜体叙事描述
 └ .ic-foot                      ← 消耗（水滴图标）/ 位置（定位图标）
```

## 六、数值 chip → 配色（.ic-stat-<名>）

- 攻击力 → 红 `#ff8a8a`
- 防御力 → 蓝 `#8ec3f0`
- 命中 / 闪避 → 金 `#e6c478`
- 修行速度 / 恢复 → 紫 `#b8a4f0`
- 灵气消耗 / 灵气受击 / 灵气容量 → 蓝 `#93c5e9`
- 加成 / 减免% / 穿透% → 绿 `#93dca5`

数值来源：`标签` 里的 `名:值` 会被解析成 chip；非 KV 的标签归为描述标签。

## 七、字段解析要点

- `标签: [攻击力:5][天品]...` → `攻击力:5` 进数值 chip；`天品` 进描述标签。
- `效果: [名: 描述][名2: 描述2]` 或单条 `名: 描述` 或纯描述（key 记为"效果"）。
- 秘籍：`完整度` / `阅读进度` 渲染为 meta mini pill。
- 傀儡/灵兽：`气血: 现值X/上限Y`、`灵气: …`、`遁速: X`、`防御力: X`、`技能` 行独立归类。
- 底栏只在 `消耗≠无` 或有 `位置` 时出现。

## 八、对初始资材的启示（设计方向）

要让"剧情物品/各类型"有辨识度，可在**保持与正文美化同源观感**（品质色带 + 类型图标 + 效果块）的基础上，为不同类型追加区分手段，候选方向：

1. **图标主题化**：直接复用 `ITEM_TYPE_ICON`，让秘籍=书、工具=扳手、神识=脑，一眼区分。
2. **按大类分色系**：功法(紫)/装备(蓝)/物品(琥珀)/傀儡灵兽(青)——头部或色带用大类色，而非仅品质色。
3. **剧情物品叠加"剧情"标识**：在同源卡上加毒紫描边 + 🔒/★剧情角标，既统一又特殊。

→ 具体样式在 Chrome 中做可视化对比后由用户拍板。
