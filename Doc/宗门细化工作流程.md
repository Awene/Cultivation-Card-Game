# 宗门细化工作流程

> 给每个宗门补一层**「门内视角」**：当玩家本人就是该宗弟子时，按其境界输出对应层级的宗门生活。
> 配套清单：[宗门设定清单.csv](宗门设定清单.csv)（27 宗门 · 主要人物 · 已有人设角色 · 主线剧情占位）
> 首个样例：**五毒教**（南疆·雨瘴林·神木殿）

---

## 一、要解决的问题

现在 `<region>总览.txt` 里的 `sectsData` 只有两档：

| 档位 | 触发 | 内容 |
|---|---|---|
| `brief` | `other` 模式（玩家在别的域） | 一句话名片 |
| `detail` | `in` 模式（玩家在本域） | 宗门客观档案：简介／理念／位置／声望／组织／地标／外销／对外 |

两档都是**外部视角**——把宗门当成一个"你会遇到的势力"来描写。但如果玩家**自己就是这个宗门的弟子**，他需要知道的是完全不同的东西：我怎么进来的？我每天干什么？我头上是谁？我下一级要跨什么坎？

所以加**第三档**：`门内`。它只在"玩家是本宗弟子"时追加输出，且**内容随玩家境界分层**。

---

## 二、变量约定

两个变量，路径和类型如下（已核对 `[initvar]` 与 `[mvu_update]变量更新规则.yaml`）：

| 变量 | 路径 | 类型 | 说明 |
|---|---|---|---|
| 身份 | `stat_data.身份` | **`string[]`（数组）** | 如 `["五毒教内门弟子"]`、`["散修"]`；可多个 |
| 境界 | `stat_data.修炼进度.境界` | `string` | 大境界+小境界，如 `"练气初期"`；凡人无小境界 |

> ⚠️ **身份是数组不是字符串**，判断必须先 `join` 或 `some`，直接 `.includes()` 会在数组上得到错误结果。

**宗门归属判定**：身份数组里任一项包含宗门关键字即可（用短词，如五毒教用 `'五毒'`，能同时命中"五毒教外门弟子""五毒使""五毒教圣女"）。

**境界 → L 值**（沿用 [世界设定-修为境界.txt](../世界书/世界设定-修为境界.txt) 的既有定义）：

| 境界 | L |
|---|---|
| 凡人 | 0 |
| 练气/炼气 | 1 |
| 筑基 | 2 |
| 金丹 | 3 |
| 元婴 | 4 |
| 化神 | 5 |

---

## 三、分档规则

| L | 档位 | 写什么 |
|---|---|---|
| 0–1（凡人／练气） | **外门** | ①**如何拜入宗门**（入门试炼的完整情节）②外门弟子／杂役的日常修炼 |
| 2（筑基） | **内门** | 内门弟子的生活：师承、当值、修炼法门、部内竞争 |
| 3（金丹） | **管事** | 管事级：管一摊事、带弟子、定配额、开始被卷入宗门政治 |
| 4–5（元婴及以上） | **长老** | 长老级：掌一部权柄、联席议事、接触宗门最深的秘密 |

### 分档必须按本宗实际组织境界校准

上表是**默认档**，不是铁律。落地前先查 [宗门设定清单.csv](宗门设定清单.csv) 的「主要人物」列，看该宗组织架构里各职位的**实际境界**，再决定分档。

- **五毒教恰好吻合默认档**：圣女蓝幽月是金丹后期（≈管事层天花板），五毒使（部长老）全是元婴，教主田殊兰化神中期。所以 金丹=管事 / 元婴=长老 完全对得上。
- **但小宗要下调**：盐镜社社主镜寒空只有金丹中期、冻骨观观主沈夜归金丹后期——在这些宗门里，**金丹就已经是掌门**，不能再写成"管事"。
- 反过来，[世界设定-修为境界.txt](../世界书/世界设定-修为境界.txt) 原文说金丹"通常担任大宗门的实权长老、一峰之主"——那是**跨宗门的普遍社会地位**。宗门内部分档以该宗自己的组织表为准，两者不冲突。

## 四、代码改法

### 4.1 在 `if (mode !== 'hidden') {` 块内、`sectsData` 之前，加公共判定

```js
  // ========== 玩家门内身份判定 ==========
  const 身份List = getMessageVar('stat_data.身份') || [];
  const 身份文本 = Array.isArray(身份List) ? 身份List.join(' ') : String(身份List);
  const 境界 = getMessageVar('stat_data.修炼进度.境界') || '凡人';
  const L = 境界.includes('化神') ? 5
          : 境界.includes('元婴') ? 4
          : 境界.includes('金丹') ? 3
          : 境界.includes('筑基') ? 2
          : (境界.includes('练气') || 境界.includes('炼气')) ? 1
          : 0;
  // 玩家是否本宗弟子: 身份数组里任一项含该宗关键字
  const 本宗弟子 = (kws) => (kws || []).some(k => 身份文本.includes(k));
  // L → 门内档位
  const 门内档 = L <= 1 ? '外门' : L === 2 ? '内门' : L === 3 ? '管事' : '长老';
```

### 4.2 在 `sectsData` 的宗门条目里加 `门内kws` 与 `门内`

```js
    '五毒教': {
      type: '大',
      门内kws: ['五毒'],          // 身份含此词 → 视为本宗弟子
      brief: `...(原样不动)...`,
      detail: `...(原样不动)...`,
      门内: {
        外门: `...`,   // L0-1: 入门情节 + 外门日常
        内门: `...`,   // L2
        管事: `...`,   // L3
        长老: `...`,   // L4-5
      }
    },
```

### 4.3 改 `'in'` 模式的宗门装配（原来是一行 map）

```js
    // 'in' 模式: 全部宗门 detail; 若玩家为本宗弟子, 按境界追加门内视角
    sectsOut = Object.values(sectsData).map(s => {
      if (!s.detail) return '';
      let out = s.detail;
      if (s.门内 && 本宗弟子(s.门内kws)) {
        out += '\n\n' + s.门内[门内档];
      }
      return out;
    }).filter(Boolean).join('\n\n');
```

### 4.4 作用范围

`门内` 只在 **`in` 模式**（玩家身处本域）输出——因为 `other` 模式本来就只渲染 `brief`，宗门 `detail` 根本不出场。也就是说：**五毒教弟子人在南疆才会看到门内视角，跑去中原就只剩其他域的一句话名片**。这符合直觉（你在总坛才过门派日常），也不额外增加他域的 token。

> 可选扩展（暂不做）：玩家**不是**本宗弟子但人在宗门所在地时，只输出「入门途径」一小段，方便他拜入。

## 五、内容规范（每档必写的东西）

每一档都是一段追加在 `detail` 之后的正文，建议 150–400 字，写成**第二人称／代入式的门内常识**，不要写成客观档案（客观部分 `detail` 已经有了，别重复）。

每档至少覆盖：

1. **你是谁** —— 该档位在宗门里的确切位置、称谓
2. **你每天干什么** —— 具体到地点（用宗门已有地标）和活计
3. **你怎么修炼／怎么变强** —— 该宗特有的修炼方式
4. **你头上是谁、身边是谁** —— 挂到已有人设角色上
5. **你的待遇与门规** —— 月例、可支取的资源、不能犯的忌
6. **下一级的坎是什么** —— 升档门槛，给玩家目标

### ⚠️ 只写生活，不写主线

`门内` 这一层**只描写宗门日常生活**：怎么进来、每天做什么、怎么修炼、谁管你、怎么升上去。

**不要在这里写主线剧情**——不埋悬念、不抖秘密、不做道德抉择、不预告冲突。宗门的主线剧情是**独立的一层**，稍后单独设计并填入 [宗门设定清单.csv](宗门设定清单.csv) 的「是否有主线剧情 / 主线剧情简介」两列，届时再决定用什么机制（剧情变量／独立条目）挂载。

生活层写干净了，主线才好往上叠；两者混在一起会导致玩家一进门就被剧透，也会让日常描写失去"日常感"。

**硬性要求：只用该宗已有的设定**——地标、组织、理念、外销、已有人设角色，全部来自 `sectsData` 与 `人物-*.txt`。不要新造地名和人名；确实需要新角色时，优先填 CSV 里**已在组织架构中、但还没有人设文件**的空位（如五毒教的吴公长老／蝎万里／毕幽影）。

## 六、五毒教样例内容

素材来源（全部为既有设定）：

- 理念：**毒亦是道，与毒共生**
- 声望：**可下毒亦可解毒，人皆敬畏**。传闻其不仅以毒虫异兽炼毒，甚至以活人为"药引"与"器皿"
- 地标：神木殿 / 百草瘴（毒草药圃） / 万毒窟 / 炼蛊池 / **万毒祖窟**
- 组织：教主 田殊兰（化神中期）／圣女 蓝幽月（金丹后期）／五毒使：天蜈部 吴公长老（元婴后期）、灵蛇部 佘青丝（元婴中期）、魔蝎部 蝎万里（元婴中期）、圣蟾部 詹金蟾（元婴初期）、鬼虎部 毕幽影（元婴初期）
- 外销：毒蛊 / 解毒丹 / 毒料 / 蛊虫殡仪服务
- 对外：道盟成员；**与千形宗达成协议可随意捉捕百越国凡人，但不可大肆屠戮**
- 秘境：**万毒祖窟**（筑基级·常驻·神木殿地下之祖蛊圣穴；四层：毒泽外林→蛊母走廊→万毒祖池→祖蛊蛰穴）

> 以下四段为**纯生活层**，不含主线剧情。

### 外门（L0–1，凡人／练气）

```
[门内视角·五毒教外门弟子]
- 入门: 五毒教不看灵根,看"毒抗"。拜山者在百草瘴外候满一日,由当值蛊头递上一碗"引蛊汤",饮下再受一次活蛊噬咬;三日内能自行走出瘴林者,收为外门。无灵根之凡人亦可入,只做药圃杂役,不列弟子名册。
- 分部: 入门后由五毒使择人分部——天蜈、灵蛇、魔蝎、圣蟾、鬼虎,五部各领一毒。分法不看你强弱,看蛊虫亲近你哪一种:相性由虫定,不由人挑。
- 日常: 卯时进百草瘴除草、采毒草、辨毒性;午后回炼蛊池刷池、喂低阶蛊虫;申时到丹房领一粒解毒丹解瘴气。丹是白给的,但须到账房记名——圣蟾部詹长老的账精确到一钱,多领少领她都清楚。
- 修炼: 授《驱虫诀》,先学不被自己养的东西咬死。外门的功夫全在"喂"字上:你喂蛊,蛊也在养你的毒抗。毒抗涨得快的,长老会多看你两眼。
- 待遇: 月例灵石三块、解毒丹按日支取、瘴林里采到的毒草可留三成自用。
- 门规: 蛊不得私带出教;不得对同门下毒;瘴林入夜不得独行。
- 升门之坎: **结本命蛊**。择一虫入体,认主则共生,不认则反噬。结成者入内门,身上会浮出毒纹。
```

### 内门（L2，筑基）

```
[门内视角·五毒教内门弟子]
- 你是谁: 本命蛊已成,毒纹随蛊而生——纹路是内门弟子的凭证,长老一眼就知道你属哪一部、养的什么虫。入五部之一,有正经师承。
- 日常: 轮值下炼蛊池喂蛊母(池水没膝,蛊母认气不认人,喂错一勺便要翻脸);随本部长老进雨瘴林深处采毒料;替教里跑外销的单子——毒蛊、解毒丹、毒料,以及"蛊虫殡仪服务"(替凡人富户以蛊净尸,此业利厚,是教中一大进项)。
- 修炼: 以身饲蛊、以毒淬体。内门修的是"共生":今日服一分新毒,明日多得一分抗性;毒纹越深,根基越稳。走得急的会伤脉,故长老常压着弟子的进度。
- 五部风气: 天蜈部人最多、最强势,吴公长老元婴后期为五使之首;灵蛇部最冷清,佘青丝长老不带徒弟只带蛇,你去回话她听完点头便走;圣蟾部最富,詹金蟾长老抠外人却不抠同门,你缺毒料找她多半准批;魔蝎部蝎万里长老好斗,门下常私下比毒;鬼虎部毕幽影长老话少,行事最隐秘。
- 圣女: 蓝幽月金丹后期,是内门弟子的头一号。新毒方多由她先试,故内门弟子对她既服气又怵。教主田殊兰对她格外纵容,教里人尽皆知。
- 待遇: 月例灵石三十块、可自领一间蛊房、每季支取一份高阶毒料。
- 升管事之坎: 能独立主一味毒方,且经得起五毒使联席验方。
```

- 你是谁: 部内蛊头、堂主一级,管一摊实务——百草瘴药圃、丹房、炼蛊池、账房、外销,五摊里领一摊。手下有内外门弟子数十。

- 待遇: 独立洞府一座、月例灵石三百、毒料支取不设限(但要记账)。
- 升长老之坎: 修为至元婴,且五毒使有空位。五席现已满,要么等一席空出,要么由教主亲自特擢。

```
[门内视角·五毒教长老]
- 你是谁: 五毒使之一,掌一部之蛊、一部之弟子、一部之毒方。联席议事时教主田殊兰居主位,五使分坐左右。

- 待遇: 一部之资任你调度;神木殿有你一间静室;教主赐"毒使令",凭令可调五部弟子。
- 更上一层(化神): 教主之位。田殊兰化神中期,现下在位。

## 七、落地步骤

对每个宗门，按顺序：

1. 查 [宗门设定清单.csv](宗门设定清单.csv)：确认该宗的组织架构、实际境界分布、已有人设角色
2. 按「三」校准分档（小宗要下调）
3. 在 `<region>总览.txt` 的 `if (mode !== 'hidden')` 块内加一次公共判定代码（**每个总览文件只需加一次**，五个宗门共用）
4. 给该宗的 `sectsData` 条目加 `门内kws` + `门内` 四档
5. 改该文件 `'in'` 模式的 `sectsOut` 装配（**每个总览文件只需改一次**）
6. 验证：抽出 JS 主体用 node 跑，分别喂 `身份=[]` 与 `身份=["五毒教内门弟子"]`×五档境界，确认输出正确、且非本宗弟子时门内块完全不出现

## 八、注意事项

- **身份是数组**：`stat_data.身份` 是 `string[]`，判定前必须 `join`/`some`。
- **关键字要短**：用 `'五毒'` 而非 `'五毒教'`，可同时命中"五毒使""五毒教外门弟子"等各种写法。
- **别和 `detail` 重复**：`detail` 是客观档案，`门内` 是主观代入。同一件事不要说两遍。
- **只用既有设定**：地标、人名、理念全部来自现有文件。要加人，优先填组织架构里已有名字但缺人设的空位。
- **生活层与主线层分开**：本工作流只产出生活层。主线剧情后续单独设计、单独挂载，不要提前混进 `门内`。
- **token 成本**：门内块只在玩家是本宗弟子且身处本域时输出，最多一个宗门的一档，约 200–400 字，可接受。
```
