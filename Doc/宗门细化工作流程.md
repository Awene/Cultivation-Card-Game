# 宗门细化工作流程

> 给每个宗门补一层**「门内视角」**：当玩家本人就是该宗弟子时，按其境界输出对应层级的宗门生活。
> 配套清单：[宗门设定清单.csv](宗门设定清单.csv)（27 宗门 · 主要人物 · 已有人设角色 · 主线剧情占位）
>
> **两步走**：
> - **第一步 · 门派修炼细化（本文档，已定稿）** —— 门内视角四档：入门途径 + 各境界的日常/本命蛊/功法书/技艺书。**只写生活与修炼，不写主线**。
> - **第二步 · 主线剧情（待做）** —— 独立一层，另行设计并挂载，填入 [宗门设定清单.csv](宗门设定清单.csv) 的「是否有主线剧情 / 主线剧情简介」两列。
>
> 标杆样例：**五毒教**（南疆·雨瘴林·神木殿），代码见 [南疆总览.txt](../世界书/凡界/南疆/南疆总览.txt) `sectsData['五毒教'].门内`。下文所有格式规范以该定稿为准。

---

## 一、要解决的问题

现在 `<region>总览.txt` 里的 `sectsData` 只有两档：

| 档位 | 触发 | 内容 |
|---|---|---|
| `brief` | `other` 模式（玩家在别的域） | 一句话名片 |
| `detail` | `in` 模式（玩家在本域） | 宗门客观档案：简介／理念／位置／声望／组织／地标／外销／对外 |

两档都是**外部视角**——把宗门当成一个"你会遇到的势力"来描写。但如果玩家**自己就是这个宗门的弟子**，他需要知道的是完全不同的东西：我怎么进来的？我每天干什么？我头上是谁？我下一级要跨什么坎？

所以加**第三档**：`门内`。它只在"玩家是本宗弟子"时追加输出，且**内容随玩家境界分层**。

---

## 二、变量约定

两个变量，路径和类型如下（已核对 `[initvar]` 与 `[mvu_update]变量更新规则.yaml`）：

| 变量 | 路径 | 类型 | 说明 |
|---|---|---|---|
| 身份 | `stat_data.身份` | **`string[]`（数组）** | 如 `["五毒教内门弟子"]`、`["散修"]`；可多个 |
| 境界 | `stat_data.修炼进度.境界` | `string` | 大境界+小境界，如 `"练气初期"`；凡人无小境界 |

> ⚠️ **身份是数组不是字符串**，判断必须先 `join` 或 `some`，直接 `.includes()` 会在数组上得到错误结果。

**宗门归属判定**：身份数组里任一项包含宗门关键字即可（用短词，如五毒教用 `'五毒'`，能同时命中"五毒教外门弟子""五毒使""五毒教圣女"）。

**境界 → L 值**（沿用 [世界设定-修为境界.txt](../世界书/世界设定-修为境界.txt) 的完整阶梯）：

| 境界 | L | 门内档 |
|---|---|---|
| 凡人 | 0 | 外门 |
| 练气/炼气 | 1 | 外门 |
| 筑基 | 2 | 内门 |
| 金丹 | 3 | 管事 |
| 元婴 | 4 | 长老 |
| 化神 | 5 | 长老 |
| 返虚（别名炼虚） | 6 | 长老 |
| 合体 | 7 | 长老 |
| 大乘 | 8 | 长老 |
| 渡劫 | 9 | 长老 |

三条健壮性约定（落地时务必遵守）：

1. **凡人与炼气同看「外门」**——设定上凡人 L=0，但门内档不再细分，凡人直接看炼气档的入门内容。
2. **化神及以上一律归「长老」**——返虚/合体/大乘/渡劫这些高阶境界必须显式识别（尤其**返虚可能被写作「炼虚」**），归入 L≥4 的长老档；绝不能因为没写进映射而掉进兜底。
3. **无法识别的境界兜底为「外门」，而非整段隐藏**——遇到没见过的境界串，默认按最低档（外门）显示，保证门内视角始终有内容。另外 `境界` 取回后**强制 `String()` 转换**，防止万一是数字类型时 `.includes` 抛错、导致整段 EJS 渲染失败。

## 三、分档规则

| L | 档位 | 写什么（均为生活+修炼层，不含主线） |
|---|---|---|
| 0–1（凡人／练气） | **外门** | 拜入试炼（入门）＋新手资源（如筑基丹）＋外门日常＋炼气级功法/技艺书 |
| 2（筑基） | **内门** | 结本命蛊、内门日常（育蛊/采料/接单）＋筑基级功法/技艺书 |
| 3（金丹） | **管事** | 领一摊实务、带徒、对账、接大单＋金丹级功法/技艺书 |
| 4–5（元婴及以上） | **长老** | 掌一部、议事分账定价、代教出席＋元婴级功法/技艺书（含顶阶禁术） |

### 分档必须按本宗实际组织境界校准

上表是**默认档**，不是铁律。落地前先查 [宗门设定清单.csv](宗门设定清单.csv) 的「主要人物」列，看该宗组织架构里各职位的**实际境界**，再决定分档。

- **五毒教恰好吻合默认档**：圣女蓝幽月是金丹后期（≈管事层天花板），五毒使（部长老）全是元婴，教主田殊兰化神中期。所以 金丹=管事 / 元婴=长老 完全对得上。
- **但小宗要下调**：盐镜社社主镜寒空只有金丹中期、冻骨观观主沈夜归金丹后期——在这些宗门里，**金丹就已经是掌门**，不能再写成"管事"。
- 反过来，[世界设定-修为境界.txt](../世界书/世界设定-修为境界.txt) 原文说金丹"通常担任大宗门的实权长老、一峰之主"——那是**跨宗门的普遍社会地位**。宗门内部分档以该宗自己的组织表为准，两者不冲突。

## 四、代码改法

### 4.1 在 `if (mode !== 'hidden') {` 块内、`sectsData` 之前，加公共判定

```js
  // ========== 玩家门内身份判定 ==========
  // stat_data.身份 是 string[] (如 ["五毒教内门弟子"]), 必须先拼成文本再判断。
  const 身份List = getMessageVar('stat_data.身份') || [];
  const 身份文本 = Array.isArray(身份List) ? 身份List.join(' ') : String(身份List);
  // 境界强制转字符串: 万一取回数字/异常值, 避免 .includes 抛错导致整段渲染失败
  const 境界 = String(getMessageVar('stat_data.修炼进度.境界') || '凡人');
  // 完整阶梯: 凡人0/炼气1/筑基2/金丹3/元婴4/化神5/返虚(炼虚)6/合体7/大乘8/渡劫9
  //   化神及以上一律归长老; 无法识别的境界兜底按外门(L=1), 不整段隐藏
  const L = 境界.includes('化神') ? 5
          : 境界.includes('元婴') ? 4
          : 境界.includes('金丹') ? 3
          : 境界.includes('筑基') ? 2
          : (境界.includes('炼气') || 境界.includes('练气')) ? 1
          : 境界.includes('凡人') ? 0
          : (境界.includes('返虚') || 境界.includes('炼虚')
             || 境界.includes('合体') || 境界.includes('大乘') || 境界.includes('渡劫')) ? 6
          : 1;
  // 玩家是否本宗弟子: 身份数组里任一项含该宗关键字
  const 本宗弟子 = (kws) => (kws || []).some(k => 身份文本.includes(k));
  // L → 门内档位: 凡人/炼气=外门, 筑基=内门, 金丹=管事, 元婴及以上=长老
  const 门内档 = L <= 1 ? '外门' : L === 2 ? '内门' : L === 3 ? '管事' : '长老';
```

### 4.2 在 `sectsData` 的宗门条目里加 `门内kws` 与 `门内`

```js
    '五毒教': {
      type: '大',
      门内kws: ['五毒'],          // 身份含此词 → 视为本宗弟子
      brief: `...(原样不动)...`,
      detail: `...(原样不动)...`,
      门内: {
        外门: `[门内视角]\n...`,   // L0-1: 入门/筑基丹来源 + 外门日常/功法书/技艺书
        内门: `[门内视角]\n...`,   // L2
        管事: `[门内视角]\n...`,   // L3
        长老: `[门内视角]\n...`,   // L4-5
      }
    },
```

### 4.3 改 `'in'` 模式的宗门装配（原来是一行 map）

```js
    // 'in' 模式: 全部宗门 detail; 若玩家为本宗弟子, 按境界追加门内视角
    sectsOut = Object.values(sectsData).map(s => {
      if (!s.detail) return '';
      let out = s.detail;
      if (s.门内 && 本宗弟子(s.门内kws)) {
        out += '\n\n' + s.门内[门内档];
      }
      return out;
    }).filter(Boolean).join('\n\n');
```

### 4.4 作用范围

`门内` 只在 **`in` 模式**（玩家身处本域）输出——因为 `other` 模式本来就只渲染 `brief`，宗门 `detail` 根本不出场。也就是说：**五毒教弟子人在南疆才会看到门内视角，跑去中原就只剩其他域的一句话名片**。这符合直觉（你在总坛才过门派日常），也不额外增加他域的 token。

> 可选扩展（暂不做）：玩家**不是**本宗弟子但人在宗门所在地时，只输出「入门途径」一小段，方便他拜入。

## 五、门内块格式规范（以五毒教定稿为准）

每一档都是一段追加在 `detail` 之后的正文，第一行固定 `[门内视角]`（**不带宗门名/档位后缀**——档位由代码按境界选中，正文不必自报是哪一档）。其后是若干 `- 条目名:内容` 行。条目分三类：

### A. 恒定条目（四档逐字相同，直接复制）

- `叙事指导:` —— 给模型的写作定调，一句话点明该宗基调，防止跑偏。五毒教是 `五毒教并非邪教，风格以中医与毒理研究为主`。**四档完全一致**。

### B. 仅外门出现的条目（入门相关，升档后删除）

- `入门:` —— 如何拜入的完整试炼情节
- `筑基丹来源:` —— 该宗给新人的一次性资源承诺（五毒教：修为达炼气后期可领筑基丹一次）
- 这类"怎么进来 / 新手福利"只写在外门，内门及以上不再出现。

### C. 随境界升级的条目（每档一份，逐级抬升）

- `本命蛊:`（或该宗的核心修炼载体）—— 外门写"还不能拥有"，内门及以上写"已拥有、随境界同步"。物品要标游戏含义：`(视为持有的灵兽类物品)`。
- `日常:` —— 该档位每天干什么，用宗门已有地标 + 职能。五毒教：外门采毒草喂蛊诊病 → 内门下池育蛊采料接单 → 管事领摊带徒对账 → 长老议事分账定价。
- `功法书:` —— 见下【功法书规范】。
- `技艺书:` —— 六类（炼丹/炼器/制符/驯兽/培育/医术）书目，逐档换更高深的书名（纯书名罗列，无品阶标注）。

### 【功法书规范】（重点）

功法书按槽位分：心法 / 幻术 / 咒法 / 神识 / 身法 / 护体，每槽 0–2 部。**不必六槽齐全**——按该档位需要取舍（五毒教内门只写了心法/幻术/身法/护体，长老只写心法/幻术/护体）。

每部写成 `《书名》(境界+品阶，机制化效果)`，遵守四条：

1. **"升级"是升境界，不是升品阶**。外门是「炼气」境界下的凡品~天品若干功法；内门换成「筑基」境界的**全新功法**（品阶重新从凡品铺开），管事「金丹」，长老「元婴」。即：原本一部炼气黄品的功法，内门对应的是**另写一部筑基凡品/黄品的新功法**——境界前缀抬高、功法名换新，而非把同一部功法的品阶往上顶。
2. **品阶体系**：凡品 < 黄品 < 玄品 < 地品 < 天品（对应 [世界设定-物品功法.txt](../世界书/世界设定-物品功法.txt) 的凡/黄/玄/地/天阶）。同一档内各功法品阶可自由分布。
3. **效果要机制化、可玩**，不要纯文学描述。照五毒教写法：`为所有敌人添加1层中毒状态`、`闪避敌人攻击时立刻结算一次中毒伤害`、`受到中毒的敌人攻击时+2闪避值`、`运功时化解丹毒速度+100%`、`每回合回复气血`、`成功闪避后赋予敌人减少幻术技艺的负面状态`。写的是能被 [战斗规则] 消费的量。
4. **禁术（可选）**：顶阶功法可设为"禁术"，需付出代价 / 完成仪式方可修炼。五毒教两部天品心法《噬灵毒龙功》《噬魂毒祖功》即禁术，需先完成"噬灵/噬魂仪式"（把活人置入蛊群异化、虫入七窍夺灵，被夺灵者视为 L+1，可重复）。

### 物品/资源的游戏化标注

凡玩家会"获得"的东西，用括号补一句游戏含义，方便变量系统与战斗判定接管：本命蛊 `(视为持有的灵兽类物品)`、`修为达到炼气后期可领取筑基丹（仅一次）`、功法效果里的 `1层中毒状态`/`+2闪避值`/`L+1` 等。

### ⚠️ 只写生活与修炼，不写主线

`门内` 这一层只描写：怎么进来、每天做什么、能修什么功法、能领什么资源、怎么升上去。

**可以陈列"存在什么"，但不展开"围绕它的故事"**——比如禁术、活人祭这类黑暗**机制**可以作为"可修炼的功法/客观存在的做法"写进来（五毒教定稿就保留了噬灵禁术，与"叙事指导：并非邪教"形成有意的张力）；但**不要在门内层叙述主线**：不埋悬念、不抖秘密、不做道德抉择、不预告冲突。谁在背地里做什么、教主知不知情、这笔账最后怎么了结——那是**第二步主线剧情**的事，另行设计、单独挂载。

生活层写干净了，主线才好往上叠；混在一起会让玩家一进门就被剧透，也让日常描写失去"日常感"。

**硬性要求：只用该宗已有的设定**——地标、组织、理念、外销、已有人设角色，全部来自 `sectsData` 与 `人物-*.txt`。功法书/技艺书的书名、以及少量为玩法服务的地标（如五毒教新增的"五圣秘浴")可自拟，但须贴合该宗主题、且不与既有设定冲突。要加人，优先填 CSV 里**已在组织架构中、但还没有人设文件**的空位（如五毒教的吴公长老／蝎万里／毕幽影）。

## 六、五毒教定稿（标杆样例）

完整内容以 [南疆总览.txt](../世界书/凡界/南疆/南疆总览.txt) `sectsData['五毒教'].门内` 为准，此处只记录其结构与升级脉络，供仿写时对照。

素材来源（全部为既有设定，已与定稿核对）：

- 理念：**毒亦是道，与毒共生**
- 声望：**可下毒亦可解毒，人皆敬畏**。传闻其不仅以毒虫异兽炼毒，甚至以敌人为"药引"与"器皿"
- 地标：神木殿 / 百草瘴（毒草药圃） / 万毒窟 / 炼蛊池 / **五圣秘浴（解蛊温泉）**
- 组织：教主 田殊兰（化神中期）／圣女 蓝幽月（金丹后期）／五毒使：天蜈部 吴公长老（元婴后期）、灵蛇部 佘青丝（元婴中期）、魔蝎部 蝎万里（元婴中期）、圣蟾部 詹金蟾（元婴初期）、鬼虎部 毕幽影（元婴初期）
- 外销：毒蛊 / 解毒丹 / 毒料 / 殡仪服务
- 对外：道盟成员；**与千形宗达成协议可随意处置百越国凡人，但不可大肆屠戮**

### 四档条目一览（√=有该条目）

| 条目 | 外门 | 内门 | 管事 | 长老 | 说明 |
|---|---|---|---|---|---|
| 叙事指导 | √ | √ | √ | √ | 四档逐字相同 |
| 本命蛊 | √ | √ | √ | √ | 外门"尚不能拥有" vs 内门+"已结、随境同步" |
| 筑基丹来源 | √ | | | | 仅外门（一次性新手资源） |
| 入门 | √ | | | | 仅外门（拜山试炼） |
| 日常 | √ | √ | √ | √ | 逐档升：采药诊病 → 育蛊接单 → 领摊带徒 → 议事分账 |
| 功法书 | √ | √ | √ | √ | 境界逐档抬升，见下 |

### 功法书升级脉络（境界抬升、非品阶抬升）

| 档 | 境界前缀 | 心法举例 | 禁术 |
|---|---|---|---|
| 外门 | 炼气 | 《驱虫诀》炼气凡品 / 《玄蛇噬灵功》炼气天品 | — |
| 内门 | 筑基 | 《化毒诀》筑基凡品 / 《天蛇功》筑基地品 | — |
| 管事 | 金丹 | 《天蛊诀》金丹地品 / 《噬灵毒龙功》金丹天品 | **噬灵仪式** |
| 长老 | 元婴 | 《化蛊诀》元婴地品 / 《噬魂毒祖功》元婴天品 | **噬魂仪式** |

- 效果全部机制化：如外门《噬灵毒雾》"为所有敌人添加1层中毒状态"、《蜈蚣幻步》"闪避敌人攻击时立刻结算一次中毒伤害"、《蛇行步》"受到中毒的敌人攻击时+2闪避值"。
- 两部天品心法为**禁术**：需先完成"噬灵/噬魂仪式"（活人置入蛊群异化、虫入七窍夺灵，被夺灵者视为 L+1，可重复）——这层黑暗与"叙事指导：并非邪教"的张力是**有意保留**的，正是第二步主线的引信，写生活层时照录、不展开。

> 提示：五毒教定稿把功法槽位按档位做了取舍（内门无咒法/神识槽，长老只留心法/幻术/护体）——**槽位齐不齐全不重要，贴合该档需要即可**。

## 七、落地步骤

对每个宗门，按顺序：

1. 查 [宗门设定清单.csv](宗门设定清单.csv)：确认该宗的组织架构、实际境界分布、已有人设角色
2. 按「三」校准分档（小宗要下调）
3. 在 `<region>总览.txt` 的 `if (mode !== 'hidden')` 块内加一次公共判定代码（**每个总览文件只需加一次**，五个宗门共用）
4. 给该宗的 `sectsData` 条目加 `门内kws` + `门内` 四档
5. 改该文件 `'in'` 模式的 `sectsOut` 装配（**每个总览文件只需改一次**）
6. 验证：抽出总览文件的 JS 主体用 node 跑（四档标题都是 `[门内视角]`，按每档独有的功法名等特征句判断命中哪一档），覆盖以下矩阵：
   - 本宗弟子 × 各境界（含返虚/炼虚/合体/大乘/渡劫、以及非法/数字境界）→ 输出且仅输出对应一档
   - 非本宗 / 空身份 → 门内块完全不出现
   - 本宗弟子但人在他域（other 模式）→ 只剩 brief、无门内块
   - 回归：既有秘境白名单（other 露出数、in 秘境段）未被破坏

   > 参考测试脚本：`scratchpad/test_menei.mjs`（含完整断言矩阵，改宗门后同步换特征句即可）。

## 八、注意事项

- **身份是数组**：`stat_data.身份` 是 `string[]`，判定前必须 `join`/`some`。
- **境界要转字符串**：`String(...)` 兜底，防数字类型 `.includes` 抛错整段炸掉；无法识别的境界兜底外门；返虚可能写作"炼虚"。
- **关键字要短**：用 `'五毒'` 而非 `'五毒教'`，可同时命中"五毒使""五毒教外门弟子"等各种写法。
- **标题不带后缀**：四档正文第一行都是 `[门内视角]`，档位由代码选中，不在正文自报。
- **别和 `detail` 重复**：`detail` 是客观档案，`门内` 是主观代入。同一件事不要说两遍。
- **只用既有设定**：地标、人名、理念来自现有文件；功法/技艺书名与少量玩法地标可自拟但须贴题。要加人，优先填组织架构里已有名字但缺人设的空位。
- **生活层与主线层分开**：本工作流只产出生活层。黑暗**机制**（禁术等）可照录，但主线**故事**留到第二步单独挂载。
- **token 成本**：门内块只在玩家是本宗弟子且身处本域时输出，最多一个宗门的一档，可接受。
