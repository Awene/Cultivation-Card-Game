# 参数声明：
  L=境界系数 Q=品质系数 R=灵根系数
  [Z%] = 匹配当前计算属性的 max(相关增益%) - max(相关减益%)
  [W] = 匹配当前计算属性的 max(相关增值) - max(相关减值)
  （注：Z%与W为动态求值，如在计算“遁速”时，仅检索与遁速相关的状态加成，绝不可代入攻击或防御类加成）

# 核心触发条件: 
  激活：
    1. 明确进入"双方"战斗
    2. 强行破阵（与阵法战斗）
  禁止：
    1. 单方面战斗（如：境界差距L>2、处置俘虏、屠杀、剿灭溃逃之敌）
    2. 非战斗（如：卷入高阶修士战斗余波、性交、社交威压）

# 第一阶段: 战况总览
  逻辑与规则:
    1. 战斗类型: 切磋/压制/死斗
    2. 回合资源: 每单位拥有1[攻击]1[动作]
    3. 集群生成: 仅任一方存在≥3个同类低级单位时，可聚合为[集群](个体×N)；阵法除外
    4. 状态展示:
      - 确定所有参战人物，及其所祭出（使用中）的法宝、灵兽、傀儡的当前状态
      - 面板中参战单位类型仅人物、法宝、灵兽、傀儡
      - 计算人物灵力，人物开战时祭出法宝时一次性消耗所需灵力，法宝获得消耗的灵力值
      - 阵法若战斗开始前存在则无需结阵
    5. 频率: 每回合开始输出一次

# 第二阶段: 行动顺序
  逻辑与规则:
    1. 公式: [遁速] × (1+[Z%])+[W]
    2. 排序: 仅人物、法宝、傀儡、灵兽参与排序，阵法排序固定为最后；面板列表顺序<user>→队友A...→敌人A...，行动序列总值从高到低，生成本回合行动轴
    3. 频率: 每回合输出一次

# 第三阶段: 攻击行动 (核心循环)
  逻辑与规则:
    1. 攻击次数: 非集群人物多段攻击视为一次合并的攻击意图（多段仅用于描述）；集群攻击次数由当前气血百分比决定: 气血≥80%→3次, 气血≥50%→2次, 气血≥30%→1次, 气血<30%→1次
    2. 意图路由(消耗1[攻击]):
      - 判定主体: <user>/法宝/灵兽/傀儡/队友，<user>可指令友方执行意图
      - 路由表:
        | 描述匹配 | 路由 | 难度 | 技艺 |
        | --- | --- | --- | --- |
        | 一般攻击/模糊描述/攻击要害/斩杀 | → 攻击检定(5.) | - | - |
        | 缴械/致残/俘获 | → 战术意图对抗 + 攻击检定(5.) | 5 | 无 |
        | 幻术功法/幻惑效果(迷惑/混乱/昏迷) | → 幻术意图对抗(跳过攻击检定) | 2 | 幻术 |
        | 神识功法/神念效果(震慑/威压/干扰) | → 神识意图对抗(跳过攻击检定) | 0 | 神识 |
        | 破阵描述 | → 破阵意图对抗(跳过攻击检定) | 0 | 阵法 |
      - 意图对抗公式:
        floor(10^攻方L+攻方[技艺]*(1+[Z%])+[W])*d20 vs floor(10^守方L+守方[技艺]*(1+[Z%])+[W])*(d20+[难度])
      - 意图结果:
        * 战术意图: 成功→施加状态，继续攻击检定; 失败→无效，继续攻击检定
        * 幻术意图: 成功→施加状态(本次行动结束); 失败→微弱干扰(本次行动结束)
        * 神识意图: 成功→施加状态 + 真实伤害Max(10^攻方L, 攻方[技艺]*(1+[Z%])+[W])(不致死，本次行动结束); 失败→微弱干扰(本次行动结束)
        * 破阵意图: 成功→对阵法灵气造成Max(10^攻方L, 攻方[技艺]*(1+[Z%])+[W])伤害(本次行动结束); 失败→无效(本次行动结束)
      - 注: 状态仅由意图施加
    3. 攻击检定(仅路由至此时执行):
      - 公式:
        * 对无法行动者遁速取0
        * floor(10^攻方L+攻方[遁速]*(1+[Z%])+[W])*(d20+[命中]) vs floor(10^守方L+守方[遁速]*(1+[Z%])+[W])*(d20+[闪避])
      - 命中取值:
        * 直接攻击/咒法攻击 [命中] = 功法命中+[W]
        * 符箓/阵法/法宝/傀儡/灵兽/其他 [命中] = 1 + floor(L/2)+[W]
      - 闪避取值:
        * 人物 [闪避] = 身法闪避+[W]
        * 法宝/傀儡/灵兽 [闪避] = 1 + floor(L/2)+[W]
        * 阵法: 0
    4. 伤害计算(仅攻击检定命中后执行，神识意图的真实伤害直接应用):
      - 基础伤害:
        * 人物直接攻击:floor(10^L + 功法[攻击力] + [炼体]×(1+L×0.2) + 武器[攻击力])*(1+[Z%])+[W]
        * 人物咒法攻击:floor(10^L + 咒法[攻击力] + [咒法]×(1+L×0.2))*(1+[Z%])+[W]
        * 法宝攻击:floor(10^L + 法宝[攻击力] + 法宝所有者[御物]×(1+法宝L×0.2))*(1+[Z%])+[W]
        * 傀儡/灵兽攻击:floor(10^L + 傀儡/灵兽[攻击力])*(1+[Z%])+[W]
        * 阵法攻击:floor(10^L + 阵法[攻击力])*(1+[Z%])+[W]
      - 五行生克系数: //守方五行以护体功法五行为准，无则为混沌
        * 非咒法攻击: 克制1.25 | 被克制0.75 | 无克制1.0
        * 咒法攻击: 克制1.5 | 被克制0.5 | 无克制1.0
      - 穿透与减免:
        * [穿透%]: 攻方L*10% + [Z%]
        * [减免%]: 守方L*10% + [Z%]
        * 总防御力: 受击位置防具[防御力]+护体功法[防御力]+[W]
        * 结算伤害: 最终伤害 = Max(0, 基础伤害 - 总防御力) × (1 + [穿透%] - [减免%]) × [五行生克系数]
      - 集群修正: 对集群单位的最终伤害 ×1.5
      - 伤害应用:
        * 人物: if 护体功法[灵力受击]: 灵力-=伤害; 溢出=max(0,-灵力); 气血-=溢出 else: 气血-=伤害
        * 法宝/阵法: 灵气-=伤害
        * 傀儡/灵兽: 气血-=伤害
      - 生死判定: 
        * 人物/傀儡/灵兽: 气血 ≤ 0 -> [死亡/毁坏]
        * 法宝/阵法: 灵气 ≤ 0 -> [停止运转]
      - 死亡处理: 若人物所有者死亡，则该人物所祭出的[阵法/法宝/傀儡]停止运转，[灵兽]失控(转为中立或敌对单位)

# 第四阶段: 战术动作(核心循环)
  逻辑与规则:
    1. 消耗1[动作]
    2. 动作类型:
      - 动作: 燃烧精血/聚气/投掷/服用丹药/使用道具等，可主动获得状态或使丹药生效
      - 结阵: 向目标阵法注入 floor(10^L+[阵法]×(1+[Z%])+[W]) 灵气; 灵气充满→阵法状态变为[已结阵]; 对已结阵阵法无效
      - 使用符箓: 视为道具发动一次额外"咒法攻击"(转用{攻击行动}模板)，支付符箓消耗灵力(灵力不足发动失败)，L=符箓境界系数，符箓[遁速]=10^L，符箓[咒法]=10^L
      - 位移: 移动到某一区域
      - 逃跑: 对抗检定，
        * 对集群追击成功视为拦下所有人；多追1->以遁速最高者为追方，N=0；1追多：此追方可参与多次判定，N=逃方目标数（集群算1目标）
        * 公式: floor(10^追方L+追方[遁速]×(1+[Z%])+[W])×(d20-N) vs floor(10^逃方L+逃方[遁速]×(1+[Z%]))×d20

# 第五阶段: 战意判定 (条件触发)
  逻辑与规则:
    1. 触发: 仅非<user>人物气血低于战斗类型对应阈值时，允许该单位做出退出战斗的行为选择
    2. 气血阈值(低于此值方可选择退出):
        | 战斗类型 | 阈值 | 说明 |
        | --- | --- | --- |
        | 切磋 | 40% | 认输/中止 |
        | 压制 | 50% | 求饶/投降 |
        | 死斗 | 10% | 求饶/自爆/元婴外逃 |

  面板模板:
  <battle_info>
    {战意判定}
    | 触发: [人物名] 气血 [当前/最大]([百分比]) < [战斗类型]阈值 [X%] |
