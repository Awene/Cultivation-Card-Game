<战斗协议>

# 核心触发条件: 仅明确进入双方战斗时激活，非战斗（如性交、社交威压、处置俘虏、单方面屠杀等）禁止触发

# 第一阶段: 战况总览
  逻辑与规则:
    1. 战斗类型: 切磋/压制/死斗
    2. 回合资源: 每单位拥有1[攻击]1[动作]
    3. 集群生成: 仅任一方存在≥3个同类低级单位时，可聚合为[集群](个体×N)；阵法除外
    4. 状态展示:
      - 必须检视出战变量，确定所有参战角色，及其所祭出的法宝、灵兽、傀儡的当前状态
      - 计算角色灵力，角色开战时祭出法宝时一次性消耗所需灵力，法宝获得消耗的灵力值
      - 阵法若非战斗开始前存在，则应进行战术动作开始结阵
    5. 频率: 每回合开始输出一次

  面板模板:
  <action_info>
    {战况总览}
    | 回合: [当前回合数] | 类型: [战斗类型] | 环境: [简述环境] |
    | [境界-五行-阵法A (结阵中or已结阵)]: 灵力 [X/X] |
    | [境界-五行-角色A/角色集群A (当前存活数/初始数)]: 气血 [X/X] | 灵力 [X/X] | 遁速[X] 御物[X] 咒法[X] 幻术[X] 阵法[X] 神识[X] 炼体[X] |

# 第二阶段: 行动顺序
  逻辑与规则:
    1. 公式: '(遁速 × (1 + %修正)) + 固定修正(多来源取最高值)'
    2. 排序: 仅角色、法宝、傀儡、灵兽参与排序，阵法排序固定为最后；面板列表顺序<user>→队友A...→敌人A...，行动序列总值从高到低，生成本回合行动轴
    3. 频率: 每回合输出一次

  面板模板:
  <action_info>
    {行动顺序}
    | [角色A]: (遁速[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [法宝A]: (敏捷[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [灵兽A]: (遁速[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [傀儡A]: (遁速[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [角色B]: (遁速[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [法宝B]: (遁速[X] × (1+[来源]修正[Z%])) + [来源]固定修正[W] = [总值] |
    | [阵法A]: 0 |
    | 序列: [角色A] -> [角色B] -> ... |
  </action_info>
  叙事指导: (绘双方迅捷与否，出手迅疾之别)

# 第三阶段: 攻击行动 (核心循环)
  逻辑与规则:
    1. 消耗1[攻击]
    2. 攻击次数: 非集群角色多段攻击视为一次合并的攻击意图（多段仅用于描述）；集群攻击次数由当前气血百分比决定: 气血≥80%→3次, 气血≥50%→2次, 气血≥30%→1次, 气血<30%→1次
    3. d20骰机制：
      - 境界差修正（默认规则）:
        * 攻方大境界 > 守方大境界（如筑基 vs 练气）：攻方掷2d20取高位值
        * 攻方大境界 < 守方大境界（如练气 vs 筑基）：攻方掷2d20取低位值
        * 攻方大境界 = 守方大境界：攻方掷1d20
        * 装备/技能若提供优/劣势骰修正，则在此基础上额外掷d20（优势劣势骰互相抵消）
      - 加值规则统一：在同一位置（命中加值/闪避加值/其他加值）的多个来源[W]或[Z%]以最大的一个值为准
    4. 意图与压制解析:
      - 判定主体: <user>/法宝/灵兽/傀儡/队友均可进行意图判定，<user>可明确指令友方执行意图，未指明时默认攻击检定
      - 条件判断:
        - 一般攻击、模糊描述(最后一击/全力攻击/砍向敌人)、攻击致命部位(眼/喉/心)、斩杀描述(斩首/吞噬)不触发，进入攻击检定
        - 意图包含缴械、致残(眼/腿/手)、俘获(活捉/捆绑/击晕)时，视为[战术意图]，[难度]=5，[技艺]=无
        - 意图包含幻术功法、幻惑效果描述(迷惑/混乱/昏迷)时，视为[幻术意图]，[难度]=2，[技艺]=幻术
        - 意图包含神识功法、神念效果描述(震慑/威压/干扰)时，视为[神识意图]，[难度]=0，[技艺]=神识
        - 意图包含破阵描述时，视为[破阵意图]，[难度]=0，[技艺]=阵法，守方[境界系数]=阵法境界
      - 公式:
        - floor(10^攻方[境界系数]+攻方[技艺]*(1+[来源]修正[Z%]))*d20 vs floor(10^守方[境界系数]+守方[技艺]*(1+[来源]修正[Z%]))*(d20+[难度])
      - [战术意图]成功->意图生效; 失败->意图无效
      - [幻术意图]成功->意图生效; 失败->施加微弱干扰
      - [神识意图]成功->意图生效且造成Max(10^攻方[境界系数],攻方[技艺]*(1+[来源]修正[Z%])+[来源]固定修正[W])的真实伤害（不致死）; 失败->施加微弱干扰
      - [破阵意图]成功->对阵法灵气造成Max(10^攻方[境界系数],攻方[技艺]*(1+[来源]修正[Z%])+[来源]固定修正[W])的伤害; 失败->意图无效
    5. 攻击检定:
      - 公式:
        - 对无法行动者遁速取0
        - floor(10^攻方[境界系数]+攻方[遁速]*(1+[来源]修正[Z%]))*(d20+[命中加值]) vs floor(10^守方[境界系数]+守方[技艺]*(1+[来源]修正[Z%]))*(d20+[闪避加值])
      - 命中加值:
        - 角色: [攻击功法/咒法]命中+[来源]固定修正[W]
        - 阵法/法宝/傀儡/灵兽: 0+[来源]固定修正[W]
      - 闪避加值:
        - 角色: [身法技能]固定闪避[W]+[来源]固定修正[W]
        - 法宝/傀儡/灵兽: 0+[来源]固定修正[W]
        - 阵法: 0
    6. 伤害计算:
      - 基础伤害:
        - 人物直接攻击:floor(10^[境界系数] + 功法伤害 + [炼体]×(1+[境界系数]×0.2) + 武器攻击力)*(1+[来源]修正[Z%])
        - 人物咒法攻击:floor(10^[境界系数] + 咒法伤害 + [咒法]×(1+[境界系数]×0.2))*(1+[来源]修正[Z%])
        - 法宝攻击:floor(10^[境界系数] + 法宝攻击力 + 法宝所有者[御物]×(1+[法宝境界系数]×0.2))*(1+[来源]修正[Z%])
        - 傀儡/灵兽攻击:floor(10^[境界系数] + 傀儡/灵兽攻击力)*(1+[来源]修正[Z%])
        - 阵法攻击:floor(10^[境界系数] + 阵法攻击力)*(1+[来源]修正[Z%])
      - 装备修正:
        - 固定数值修正: 武器/装备的[攻击力/防御力]直接加减
        - 百分比修正: 武器/装备的[攻击%/防御%]作为系数应用 → 基础伤害 × (1 + 攻击%) / [防御方气血] × (1 + 防御%)
      - 五行生克修正:
        * 普通技能/炼体攻击: 若技能五行克制受击方五行，伤害×1.25；若被克制，伤害×0.75
        * 咒法攻击: 若咒法五行克制受击方五行，伤害×1.5；若被克制，伤害×0.5
      - 穿透与减免（与境界系数挂钩）:
        * 百分比穿透: [穿透%] = min(攻方[境界系数]×10%, 80%) [防守方有穿透时、上限80%]
        * 有效防御: 对方防御 × (1 - 百分比穿透)
        * 百分比减免: [减免%] = min(防守方[境界系数]×8%, 60%) [下限0%、上限60%]
        * 结算伤害: 基础伤害 × (1 - 百分比减免) × (有效防御系数)
      - 集群修正: 对集群单位的最终伤害 ×1.5
      - 生死判定: 最终结算伤害后，若目标气血 > 0则判为[未死/存活]；气血 ≤ 0则判为[死亡/毁坏]
      - 法宝阵法判定: 最终结算伤害后，若目标气血 > 0则判为[未死/存活]；气血 ≤ 0则判为[死亡/毁坏]
    7. 范围技能处理:
        - 对多个独立目标: 若技能标签为`范围:x`，索敌x个单位，每个目标独立检定(各自掷骰→评级→伤害)
        - 对集群单位: 对集群总伤害 = `修正后的单体伤害 × min(技能标签范围x, 集群当前数量n)`
    8. 状态(不限数量):
        - 附加效果触发/仅层级压制通过的战术or意图施加:
          任意暴击(≥20): 必触发
          有效/勉强: (攻方属性+d20) vs (守方属性+d20) 对抗检定
          擦伤/失手: 不触发
        - 控制类状态加固: 施加控制类效果(禁锢/沉默/眩晕/冰冻等限制行动的状态)时，守方对抗检定额外+5

  面板模板:
  <action_info>
    {攻击行动}
    | 攻方: [攻击者] | 守方: [目标] | 招式: [技能名] | 所使武器: [装备名] |
    | 消耗: HP [X] | MP [X] | SP [X] |
    | 意图: [描述关键词] -> 判定为 [非致死/常规/机能/核心/抹杀/概念/处决] |
    {伤害基准计算}
    | 初始伤害: ([关联属性]×10×[层级系数] + [技能威力] + [武器攻击]) = [总伤害值] |
    | 初始伤害: [道具固定伤害及其它修正] = [总伤害值] | (仅伤害型道具时，不计算公式)

    | [伤害类型]伤害(占比%): 构成[A] - 装备减免([装备名] B) - 属性减免(C) = [结算基值 D] |

    | 意图判定: (攻方[X]层×5+d20[A]=[M]) vs (守方[Y]层×5+d20[B]+意图难度[C]=[N]) -> [成功/失败/自动成功/强制无效/无需判定] |
    {结算清单}
    | 掷骰: [优势/劣势:d20(骰值1,骰值2)->取值 / 正常:d20(骰值)] |
    | 命中加值: max(技能命中[a1], 装备命中[a2]) = [b] | (多来源取最高值，不叠加)
    | 闪避减值: max(目标装备闪避[c1], 目标技能闪避[c2]) = [c] | (仅守方闪避，多来源取最高值，不叠加)
    | 检定结果: d20[取值或骰值] + 命中[b] - 闪避[c] = [总值] | 结果: [评级] |
    | 伤害修正: ((基础[J] × 评级系数[K] × 意图系数[L]) + 额外固定伤害[M]) × 攻击次数[N] = [修正后伤害 M] |
    | 效果修正: [修正后伤害 M] × [伤害类型]修正 × 机制修正 = [修正后伤害 N] |
    | DR修正: [修正后伤害 N] × (1-DR[X]%) = [DR后伤害] | (仅守方有DR效果时)
    | 集群修正: [N/DR后] × 1.5 = [对集群伤害 P] | (仅目标为集群)
    | 范围结算: [P] × min(范围[x], 集群数[n]) = [最终总伤害 Q] | (仅范围技能vs集群)
    | 最终伤害: [N/DR后 或 P 或 Q] | 变更: [目标] HP [旧值] -> [新值][当前HP百分比] (-[最终伤害]) |
    | 集群减员: [存活数/初始数] (按HP百分比折算存活个体) | (仅目标为集群)
    | 是否存活: [新值] > 0 -> [未死/存活] | [新值] ≤ 0 -> [死亡/毁坏] | (非致死且评级≤有效: HP锁定1->[昏迷]) |
    | 状态施加([效果名]): [暴击/对抗检定(攻方X vs 守方Y)] -> [触发成功/失败] |

    | -> [目标A]: 检定([骰->评级]) | 伤害: ([J]×[系数]=[最终值]) | HP: [旧->新] | 效果: [触发/无] |
  </action_info>
  叙事指导: (描摹攻伐之术、招式细节、打击之感、受者之反应与伤痕)

# 第四阶段: 战术动作(核心循环)
  逻辑与规则:
    1. 消耗1[动作]
    2. 类型: 使用道具/[动作]技能/装备能力/移动/格挡/专注/其它
    3. 伤害道具: 若使用造成伤害的道具，必须转用{攻击行动}模板，初始伤害来源为固定道具伤害
    4. 逃跑: 逃跑是特殊的战术动作，需进行对抗检定，成功则战斗结束

  面板模板 (非伤害性):

  </action_info>

  叙事指导: (描摹战术，如服药动作、咒语咏唱、走位与防御之姿)

# 第五阶段: 战意判定 (条件触发)
  逻辑与规则:
    1. 触发: 仅非<user>单位HP低于其战斗类型的战意阈值时触发
    2. 阈值与判定:
        - 无需检定(自动[战意动摇]): 切磋(40%)/竞技(30%)/压制(50%)
        - 需要检定(d20<12触发[丧失战意/崩溃]): 死斗(10%)/标准(30%)/守卫(35%)
    3. 结果池: 投降, 认输, 求饶, 溃逃, 撤退, 被击昏, 被俘虏, 中止战斗, 阵线溃散, 内讧等 (不包括死亡)
    4. 处决选项: 战意动摇/崩溃触发结果池行为后，<user>可选择接受(投降/俘虏等)或处决目标(适用攻击检定中的处决规则)

  面板模板:
  <action_info>
    {战意判定}
    | 触发: [角色名] HP [当前/最大] < 阈值 [XX%] |
    | 类型: [战斗类型] -> [无需检定 / 需要检定] |
    | 判定: [自动触发 / d20=[X] vs 12] -> [战意动摇 / 崩溃 / 坚守] |
    | 结果: [角色名] [从结果池选择的行为] |
  </action_info>
  叙事指导: (触发时描摹。述敌手撤退、道心崩毁、求饶、逃窜)

# 第六阶段: 战斗结算 (战斗结束时)
  逻辑与规则:
    1. 触发: 只有一方全灭或成功逃跑/投降时输出
    2. 经验值(EXP): 严格依据<经验值获取规则>计算，逃跑/投降无EXP
    3. 命运点(FP): 根据战斗难度和表现获得
    4. 状态结算: 依据<状态规则>进行战斗→非战斗状态转化
    5. 战利品: 仅<user>亲自击败敌人时根据战斗情况决定是否有战利品（如敌人被彻底消灭/本身无财物等则无战利品），使用itemThink机制生成；若非<user>击败（如NPC击败/仅协助观战/敌人被架走等）则无
    4. 灵力回收: 角色收回祭出的法宝恢复其中剩余灵力值

  面板模板 (战斗结算):
